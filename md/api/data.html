    # 数据

    数据类型列表

    # 书籍分类

    class NovelCategory:
    category: str = "" # 分类名字
    category_int: int = 1 # 分类的 id
    sub_category : str = "" # 子分类名字
    sub_category_int : int = 1 # 子分类 id
    sort_number: int = 1 # 分类显示顺序
    sub_sort_number: int = 1 # 子分类显示顺序
    language:str = "" # 什么语言的分类

    关于分类信息通过 init 接口获得，在 book 数据中，存储的是 category_int，所以，前段需要根据 category_int 值根据 /init 接口返回的信息，获知这本书是什么类别。

    属性 类型 描述
    sub_category_int int 长度一共8位
    - 前 3 位是语言类型，从100开始，中 2 位是该语言下的一级分类，后3位是一次分类下的二级分类

    # 书籍

    class Book:
    title: str = "" # 书籍名字
    author: str = "" # 作者
    up_id: str = "" # 上传者 ID
    language: str = "" # 语言
    book64: str = "" # 书籍 ID
    subbook64: str = "" # 书籍真实 ID
    description: str = "" # 书籍描述
    cover_url: str = "" # 书籍封面
    categories: int = 0 # 书籍的章节数量
    category_int: int = -1 # 书籍的分类 ID
    status: str = "" # 书籍状态
    chapters: list[str] = None # 书籍的章节标题
    created_at: datetime = ""
    updated_at: datetime = ""

    假设一个书籍名字为《遮天》，那么 book64 就是这本书的唯一 ID，但是，这本书可以有多个版本，比如不同语言的版本，不同上传者的版本。

    那么，每一个版本对应的 id 就是 subbook64，识别一本书的唯一 id 也是通过 subbook64

    # 章节

    class NovelChapter:
    subbook64: str = "" # 和 book 中 subbook64 是对应的
    title: str = "" # 本章的名字
    content: str = "" # 本章的内容
    sort_number: int = 0 # 章节正序的下标
    chapter_title: str = "" # 卷名
    chapter_number: int = 0 # 卷的下标
    word_count: int = 0  # 字数
    created_at: datetime = ""
    updated_at: datetime = ""

    这里说一下卷名，一本书可能分为好几个卷，所以 chapter_title 表示的是本卷的名字，而 chapter_number 表示的是，在这个卷中，章节正序的下标
